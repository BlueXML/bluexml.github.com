Presentation

The purpose of reverse is to read some Alfresco configuration files and produce resulting SIDE models. With such models, it's then possible to extend and use existing alfresco objects into custom SIDE models.

Technical

  • 100% Java
  • Reading XML alfresco files using xsd schema, JAXB code generation
  • Writing SIDE models using Ecore API (so all serialization details are ignored)

Algorithmic

  • Sorting XML files according to their dependencies, so we can read them without trouble (ie missing references)
  • Optionally load in registry existing SIDE models (already reversed alfresco native models for example)
  • Read Alfresco files according to computed order
  • For each matching object:
  • Create corresponding SIDE model object
  • Add it to registry
  • Registry is used to resolve references

Reverse From Alfresco Content models

The easiest case to reverse is to have full bijective matching between each meta-model concept. At this time, the only unsupported feature is the reverse of "overrides" attributes definition. Some primitive types are not reversed as expected.

side.reverse Ant Task

Reverse can be executed in ANT targets using side.reverse task (see SIDE Ant Task).
