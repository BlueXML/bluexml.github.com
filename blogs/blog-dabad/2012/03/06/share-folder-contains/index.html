This page explains how to define a custom folder, and add the capability on create Forms to upload child for this new folder.

SIDE Project

Use the New Project wizard to create a new SIDE project, choose alfresco-3.4.6

DataModel

Extends from cm:Folder

  • Create your custom Folder 'MyFolder'
  • Import alfresco content model
  • Add generalization from 'MyFolder' to cm:Folder
  • On 'MyFolder' add metainfo single-child value=true, this enable a policy that 'move' new children from the tmp folder to 'MyFolder'

Forms Models

Initialize models

  • Right click on SIDE project
  • Clock on Initialize (or select your application model)

Create popup.form

This form model aims to define forms that can be called as share popup from another form

  • Create new form model file 'popup.form'
  • Add a new form with dataType equals to cm:content (load resource cmcontent.dt)
  • Initialize the form
  • Add template metainfo value=/side/templates/share-popup-tab-edit-form.ftl
  • Transform the content field into FileField.

Create toolbar.form

We need a dedicated form collection to use to create new document and folder, this form collection will feed the "Create new ..." menu (see ¤portal)

  • Create a new 'toolbar.form'
  • Add a new form with dataType equals to 'MyFolder'
  • Initialize the form
  • Select the ModelChoiceField contains (the child association to sys:base)
  • Set Association to the 'Content' popup form
  • Set widget to Select
  • Add metainfo 'targetDestination' value can be a nodeRef or a xpath that point to a Folder, this folder will be used to create the child for example '/app:company_home/st:sites/{site}/cm:tmp'

You can disable selection of existing contents:

  • add 'hideSelector' metainfo value=true

Portal model

Setup documentLibrary page to add new types in 'Create content ...' menu

open Portal model create a GeneratedPortlet, edit and select the toolbar form collection create a Portlet "toolbar-create-content" link the portlet to the GeneratedProtlet and to the documentLibrary page edit the page->partlet link to create layout information and have a valid model
