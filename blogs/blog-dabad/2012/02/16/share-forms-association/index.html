Simple Select

This widget is used if you choice "Select" as widget, you can also set template metainfo value=/side/controls/association-select.ftl

Metainfo options

Available option are (use metainfo to set them):

  • searchInSite [true|false]: default value "true", enable/disable filter available items on current site.
  • itemType [< prefixedQName>]: default value is set to association endPoint, can be used to select a subtype.
  • multipleSelectMode [true|false]: default value is set according to endPoint. so you can force a multiple endePoint to false so only one item can be selected, but the other alternative (single endPoint forced to multiple) will implies runtime exception
  • filterTerm [String, ]: default value ' ' (accept all value), can be used to filter items on cm:name

  • advancedQuery [< query>]: should be follow the grammar, please not ' char is used to indicate terminal items

  • < query>=< subquery>+

  • < subquery>=[' ( ']?< attributeQuery>[' '< operator>' '< attributeQuery>]*[' ) ']?
  • < attributeQuery>=< prefixedQName>'='< value>
  • < operator>=['AND'|'OR'|'NOT']
  • < value>= any String, beware of ' ' char a this moment this should cause some trouble because ' ' is use as separator
  • example :' ( cm:name=toto OR cm:name=titi ) AND cm:title=super'
  • maxResults [integer]: default -1 (unbounded), can be used to limit items numbers
  • hideSelector [true|false]: default is false, can be used to disable the select, could be used in conjunction with "add new", so user can only create new item instead of select existing ones
  • getDataSource [js function]: must be a javascript function that build a YUI DataSource to replace the default one, use this method if you can't filter as you wish using filterTerm and advancedQuery, so you can implements a new data webscript that give you same data structure in response.
  • association form class, allow the "Add" and "Edit" features , so user can create/edit a item to select.
  • To disable Add feature create metadata key=allowToCreate, value=false
  • To disable Edit feature create metadata key=allowToEdit, value=false
  • for association that allow multiple target :
  • nodoubleValue : control that added value do not already exists in list.
  • nodoubleLabel : control that added label (the displayed value cm:name in main cases) do not already exists in list.

Autocomplete

To use this widget, create a metainfo element under the modelChoiceField :

  • key=template
  • value=/side/controls/association-autocomplete.ftl
