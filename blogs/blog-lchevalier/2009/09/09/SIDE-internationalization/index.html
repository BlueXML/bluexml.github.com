Afin d'internationaliser S-IDE nous utilisons un système basé sur org.eclipse.osgi.util.NLS.

Fichiers de traduction

Les fichiers de traductions seront placés dans votre projet, ils peuvent être généré automatiquement via eclipse en faisant bouton droit sur le code source puis : Source > Externalize String (et en suivant les instructions). Chaque fichier de traduction pourra être décliné en autant de langue et spécificité géographique voulue (français de Belgique, Suisse, Quebec ou France par exemple). Par exemple pour la langue française, avec spécificité géographie à la France les fichiers possibles seront recherché dans cet ordre :

  • messages_fr_FR.properties
  • messages_fr.properties
  • messages.properties

Chaque ligne contiendra donc une clé et sa traduction :

  • ma.cle=Mon message

Il est également possible d'y mettre des paramètres :

  • ma.cle2=Mon message pour {0}

Classe commune

Une classe commune située dans le module com.bluexml.side.Util.libs et nommée Messages permet de gérer les traduction; en donnant une clé il va aller chercher la traduction correspondante pour la langue (LOCALE) de l'utilisateur (dans le sens vu plus avant).

Utilisation

Cette classe doit être instanciée une fois par plug-in, pour cela dans la classe Activator de votre plugin ajouter cette ligne en la personnalisant : public static final Messages Messages = new Messages(PLUGIN_ID, "com.bluexml.side.application.ui.action.messages"); Le second argument est le path vers vos fichier de properties.

Ensuite dans les classe de ce plugin vous pouvez appellez vos méthode ainsi :

  • Sans paramètre :

    • Activator.Messages.getString("ma.cle");

  • Avec un paramètre :

    • Activator.Messages.getString("ma.cle2",MonObjet.getMonAttribut());

  • Avec plusieurs paramètres :

    • Activator.Messages.getString("ma.cle3",new Object[] {MonObjet.getMonAttribut(),MonObject.getMonAutreAttribut()});
